## Шаг 4

Для триммирования запустили Trimmomatic в парном режиме с параметрами:
1) обрезка с 5'-конца при качестве <20
2) обрезка с 3'-конца при качестве <20
3) скользящее окно 10: среднее качество <20 → обрезка
4) отбрасывать риды короче 20 нт

```shell
TrimmomaticPE amp_res_1.fastq amp_res_2.fastq \
    reads_1P.fq reads_1U.fq \
    reads_2P.fq reads_2U.fq \
    LEADING:20 TRAILING:20 SLIDINGWINDOW:10:20 MINLEN:20
```
После выполнения операции повторно проконтролировали качество:
```shell
fastqc -o . reads_1P.fq reads_2P.fq
```

## Шаг 5

Проиндексировали референсный геном для быстрого поиска соответствий:
```shell
bwa index GCF_000005845.2_ASM584v2_genomic.fna
```
Выполнили выравнивание парных ридов, используя алгоритм BWA-MEM:
```shell
bwa mem GCF_000005845.2_ASM584v2_genomic.fna reads_1P.fq reads_2P.fq > alignment.sam
```
Проверили процент успешно выровненных ридов, конвертировав SAM в BAM:
```shell
samtools view -S -b alignment.sam > alignment.bam
samtools flagstat alignment.bam
```

99.87% ридов успешно выровнялись, 99.56% – правильно спарены (properly paired), что говорит о высоком качестве данных и отсутствии значимой контаминации.


## Шаг 6

Посредством samtools создадим mpileup файл, который позволит в дальнейшем найти варианты:
```shell
samtools mpileup -f GCF_000005845.2_ASM584v2_genomic.fna alignment_sorted.bam > my.mpileup
```
Variant Calling провели следующим образом на основе файла, полученного ранее в VarScan v2.4.6:
```shell
varscan mpileup2snp my.mpileup --min-var-freq 0.2 --variants --output-vcf 1 > VarScan_results.vcf
```
> [!NOTE]
> Параметр --min-var-freq в 20% выбран на основе анализа литературных данных. Такое значение позволит получить максимум информации, при этом позволит избежать попадания в итоговую выборку ложноположительных результатов, возникших, например, в результате ошибок секвенирования.

Проконтролировали содержимое файла:
```shell
# Первые 20 строк файла (включая заголовки)
head -n 20 VarScan_results.vcf

# Также просмотрим только строки с вариантами без мета-информации
grep -v "^##" VarScan_results.vcf | head -n 10
```

## Шаг 8

Для автоматической аннотации выявленных мутаций (тип мутации, влияние на белок, расположение относительно генов) использовали SnpEff  5.3.
SnpEff требует структуру каталогов и конфигурационный файл. В рабочем каталоге создали необходимые файлы и подкаталоги:

```shell
# Конфиг
echo "k12.genome : ecoli_K12" > snpEff.config

# Подкаталог для базы
mkdir -p data/k12

# GenBank-файл референса, переименовали в genes.gbk
cp GCF_000005845.2_ASM584v2_genomic.gbff data/k12/genes.gbk

# Построение базы данных
snpEff build -genbank -v -c snpEff.config k12
```

Запустили SnpEff в режиме ann (annotation) на файле VarScan_results.vcf, полученном на шаге 7:

```
snpEff ann -c snpEff.config k12 VarScan_results.vcf > VarScan_results_annotated.vcf
```

Антибиотикорезистентность обеспечивается тремя генами:

1. **`ftsI` (gene-b0084)** — изменение мишени (PBP3), антибиотик не связывается.
2. **`acrB` (gene-b0462)** — усиление эффлюксного насоса, антибиотик выкачивается из клетки.
3. **`envZ` (gene-b3404)** — снижение проницаемости (регуляция поринов), антибиотик хуже проникает.

Мутация в `rsgA` (gene-b4161) не вносит вклада в резистентность.
